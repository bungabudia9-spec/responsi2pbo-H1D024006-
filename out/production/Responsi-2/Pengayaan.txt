Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa itu abstract class dan Kenapa kita membutuhkan Abstarct class?
2. Apa perbedaan utama antara abstract class dan class biasa dalam pemrograman berorientasi objek? 
3. Mengapa subclass wajib mengimplementasikan semua abstract method dari parent class?
4. Mengapa abstract method tidak boleh memiliki isi implementasi di dalam abstract class?
5. Bagaimana abstract class dapat mendukung konsep polimorfisme, dan mengapa objek yang direferensikan melalui tipe abstract class tetap dapat menjalankan perilaku berbeda pada masing-masing subclass?



1. Abstract class adalah class yang tidak bisa langsung dibuat objeknya dan bisa memiliki abstract method. Kita membutuhkannya untuk membuat kerangka dasar bagi subclass agar perilaku tertentu wajib diimplementasikan.

2. Perbedaan utama: abstract class bisa memiliki abstract method tanpa implementasi, sedangkan class biasa semua method harus sudah diimplementasikan. Abstract class juga tidak bisa langsung dibuat objek.

3. Subclass wajib mengimplementasikan semua abstract method agar memiliki perilaku spesifik, sesuai kontrak yang ditentukan parent class.

4. Abstract method tidak memiliki isi karena tujuannya hanya sebagai template; implementasi detail diserahkan pada subclass yang berbeda-beda.

5. Abstract class mendukung polimorfisme karena objek bisa direferensikan melalui tipe abstract class. Saat memanggil method override, Java melakukan dynamic dispatch, sehingga setiap objek menjalankan perilaku spesifik subclass-nya walaupun direferensikan sebagai tipe parent.
