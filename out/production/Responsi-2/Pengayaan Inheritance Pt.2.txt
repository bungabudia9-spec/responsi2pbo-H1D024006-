[PERTANYAAN PENGAYAAN]

1. Mengapa constructor di class Member wajib memanggil constructor Customer menggunakan super(...)?

2. Apa tujuan memanggil super.tampilkanInfo() saat melakukan overriding method di subclass?

3. Berikan contoh kondisi di mana pemanggilan super.method() sangat penting dalam pewarisan.

4. Bagaimana keyword super membantu membedakan atribut yang memiliki nama sama antara parent dan child?

5. Mengapa pendekatan inheritance lebih efisien dibanding menyalin ulang seluruh atribut dan method ke dalam class Member?


1. Kenapa constructor Member wajib memanggil super(...)
Supaya atribut dasar dari parent Customer (nama, idCustomer, totalBelanja) langsung diinisialisasi saat membuat objek Member. Tanpa super(...), atribut parent bisa tidak terbentuk dengan benar.

2. Tujuan super.tampilkanInfo() di overriding
Agar method di subclass tetap menampilkan info dasar parent, baru menambahkan data tambahan di subclass (misal poin reward dan level).

3. Contoh pentingnya super.method()
Jika kita punya class Karyawan dan subclass Manajer, kita ingin menampilkan info dasar karyawan dulu, lalu tambahan tunjangan. Memanggil super.tampilInfo() memastikan data dasar tidak perlu ditulis ulang.

4. Super untuk membedakan atribut nama sama
Jika parent dan child punya atribut dengan nama sama, super.nama merujuk ke atribut parent, sedangkan this.nama ke atribut child. Jadi tidak terjadi konflik.

5. Mengapa inheritance lebih efisien
Karena subclass mewarisi kode parent sehingga kita tidak perlu menulis ulang atribut atau method. Ini mengurangi duplikasi kode dan memudahkan pemeliharaan.
