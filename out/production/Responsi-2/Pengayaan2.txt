Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

Apa yang dimaksud dengan constructor overloading dalam pemrograman berorientasi objek?
Bagaimana cara Java membedakan constructor yang memiliki nama sama namun parameter berbeda?
Apa perbedaan antara mengakses variabel secara langsung (misalnya: objek.variabel = nilai) dengan menggunakan method setter?
Jelaskan konsep encapsulation dan bagaimana method dapat membantu melindungi data dalam sebuah class!
Mengapa constructor tidak memiliki return type, bahkan tidak menggunakan kata kunci void?

Jawaban

1. Constructor overloading
Membuat beberapa constructor dengan nama sama tapi parameter berbeda supaya objek bisa dibuat dengan cara yang berbeda.

2. Cara Java membedakan constructor
Java lihat jumlah dan tipe parameter untuk tahu constructor mana yang dipakai.

3. Variabel langsung vs setter
Setter bisa memberi kontrol atau validasi sebelum nilai diubah, variabel langsung langsung berubah tanpa proteksi.

4. Encapsulation
Menyembunyikan data dalam class dan mengaksesnya lewat method supaya data aman dan valid.

5. Constructor tanpa return type
Constructor membuat objek, bukan mengembalikan nilai, jadi tidak perlu return type.
